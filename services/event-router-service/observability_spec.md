# Event Router Service Observability Spec

## Scope
This spec defines minimum production observability for `@mind/event-router-service`.

## Ingress Telemetry
- Health endpoint: `GET /v1/health`
- Ingress endpoint: `POST /v1/events`
- Required structured fields on all event logs:
  - `at`
  - `event_id` (when available)
  - `event_type` (when available)
  - `source`
  - `status` or `kind`
  - `latency_ms` (for accepted/failed dispatch)

## Log Streams
- `event_router_service_ingress.jsonl`
  - Every accepted event after auth/rate/schema/guardrails pass.
- `event_router_service_outcomes.jsonl`
  - Dispatch outcome (`accepted` or `dispatch_failed`) and latency.
- `event_router_service_security.jsonl`
  - Auth failures, rate limits, replay, duplicates, circuit actions, schema invalid.

## Golden Signals
1. Availability
- `health_ok_rate`
- `ingress_accept_rate`

2. Latency
- `p50`, `p95`, `p99` from `latency_ms` in outcomes

3. Reliability
- `dispatch_failed_count`
- `proof_verified_rate` (from runtime metrics pipeline)
- `allow_vs_block_ratio` (from decision log)

4. Security
- `auth_rejected_count`
- `rate_limited_count`
- `replay_rejected_count`
- `duplicate_suppressed_count`
- `guardrail_violation_count`
- `circuit_open_count`

## Alert Rules (minimum)
- Critical
  - `dispatch_failed_count > 0` for 5m on production
  - `circuit_open_count >= 1`
  - `proof_verified_rate < 0.95` on critical flow window
- Warning
  - `p95_latency_ms > 2000`
  - `ingress_accept_rate < 0.9`
  - `replay_rejected_count` trending upward for 15m

## Correlation and Traceability
- `event_id` is the canonical correlation key across:
  - ingress log
  - outcome log
  - trigger outcomes
  - risk register updates
  - review queue entries
- Every incident report must reference at least one `event_id`.

## Artifacts and Reporting
- Runtime metrics snapshot generated by:
  - `pnpm spec:runtime-metrics`
- Required reporting cadence:
  - per deploy
  - daily operational summary
  - incident postmortem on P0/P1

## Dashboard Panels (minimum)
1. Ingress Volume by `event_type`
2. Allow vs Block
3. Proof Verification Rate
4. Security Rejections by `kind`
5. Circuit Breaker Opens
6. Top Recurring Risks
7. Context Update Rate
